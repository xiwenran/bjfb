## 蚁小二官方 API 改造
- 项目已切换为官方 API Key 鉴权模式
- `src/publisher.js` 中 `ensureLogin` 现在强制要求 `apiKey`
- 不再回退用户名密码登录链路
- 若配置了 `clientId`，发布时自动走官方本机发布；否则走 cloud 发布
- 已做只读验证：可通过官方 API Key 拉取已授权账号列表

## 背景配乐实现
- 当前需求中的“抖音配音”实际指背景配乐，不是 AI 人声配音
- 项目支持通过官方音乐搜索接口查歌
- 可将所选音乐保存为默认模板（`config.json > defaultMusic`）
- 后续抖音发布默认复用这首模板音乐
- 该实现走官方 `music` 字段，符合官方接口设计
- 已新增试听能力：使用官方音乐接口返回的 `playUrl`
- 已新增弹窗音乐库：支持推荐/最近热歌/分类浏览、搜索、试听、使用
- 顶部音乐区已调整为以“打开音乐库”为主入口，避免内联搜索直接报未找到影响体验

## 飞书图片上传链路
- 项目仍保持“飞书附件下载到本地临时目录 → 蚁小二官方上传地址 → 正式发布”
- 这意味着飞书表格中的图片会先落地，再上传到蚁小二云存储
- 不使用占位符图片链接
- 小红书比特浏览器链路也复用同一份飞书素材下载结果：先落地本地图片，再由浏览器自动化上传到小红书发布页

## 小红书双渠道发布
- 小红书现在支持两种发布渠道：
  - `蚁小二`
  - `比特浏览器`
- 抖音仍固定走蚁小二
- 飞书需要新增字段 `小红书发布渠道`
  - 取值：`蚁小二` / `比特浏览器`
  - 若字段为空，项目默认按 `蚁小二` 处理
- 当前项目已新增：
  - `src/bitbrowser.js`：封装 BitBrowser 本地 API 打开/关闭 profile
  - `src/bitbrowser-xhs.js`：封装小红书发布页自动化
  - `config.json > bitbrowser`：保存 BitBrowser API 地址、发布页 URL、账号到 browserId 的映射
- 页面已新增“比特浏览器账号管理”面板：
  - 自动扫描飞书记录里出现过的小红书账号
  - 显示每个账号的记录数量与“已选择比特浏览器”的记录数
  - 用户只需在页面填写 `browserId` 并保存，配置会自动写回 `config.json`
  - 点击“同步飞书账号”时，还会同步飞书表中的两个单选字段选项：
    - `小红书账号`
    - `小红书发布渠道`
  - 同步 `小红书账号` 时会保留飞书字段中原本已有的单选选项，避免只剩当前少量记录里的账号
  - 飞书字段列表读取已补成分页，避免字段较多时只读到第一页，导致账号单选项同步不完整
- 现阶段仅小红书支持比特浏览器，抖音不走浏览器自动化
- 若一轮任务里仅有“小红书走比特浏览器”的记录，调度器不会强制先连接蚁小二

## 防重复发布增强
- 修复了音乐搜索接口调用错误，避免关键词搜索始终返回同一批歌曲
- 修复了默认配乐未真正进入自动发布链路的问题
- 增强同一飞书记录对同一平台的防重：
  - 进程内缓存
  - 飞书平台状态字段
  - 飞书备注成功/失败轨迹
  - 本地持久化账本 `publish-ledger.json`
- 目标约束：同一条飞书记录不能再次发到同一平台的不同账号
- 但“是否允许重发”的最终开关已经切到飞书平台状态字段：
  - 若手动改回“待发布”，发布器会自动清理该平台的本地账本锁
  - 这样记录会重新上传到蚁小二，而不是被旧备注/旧账本永久跳过
- 新增发布阶段状态展示：
  - 后端维护当前发布进度
  - 前端单独显示“检查飞书 / 下载素材 / 上传提交 / 发布完成”等阶段
  - 通过 SSE 实时刷新，不再只看日志猜阶段
- 前端性能优化：
  - 避免重复创建 SSE 连接
  - `/api/status` 返回未变化时跳过重渲染
  - 飞书记录列表默认按 80 条分批渲染，可点击“显示更多”
  - 目标是减少大数据量时浏览器卡顿或崩溃
  - 记录列表现已按待发布优先排序，已完成记录默认折叠显示
- 标签链路改为双保险：
  - 官方发布参数继续传 `tags`
  - 后续已撤销“标签拼进正文”的兜底方式，避免污染正文与产生重复标签
  - 飞书标签字段已改为按对象/多选结构解析，修复 `[object Object]` 混入标签的问题
  - 发布前按平台限制筛选标签：抖音取前 5 个，小红书取前 10 个，先去重后按标题/学科相关性排序
  - 前端记录卡片会直接显示“小红书计划提交标签 / 抖音计划提交标签”预览
- 默认配乐增强：
  - 页面会校验当前默认配乐是否还能在官方音乐库命中
  - 若默认配乐失效，发布时自动回退到热门纯音乐/轻音乐
  - 回退结果会写入运行日志和飞书备注，方便追踪
- 标题长度保护：
  - 小红书、抖音发布前会自动按 20 字截断标题
  - 避免因平台标题超长限制导致整个任务失败
  - 截断行为会写入运行日志与飞书备注
  - 比特浏览器小红书链路不再对“点击发布”直接判成功：
    - 找不到话题标签入口会直接报错，避免静默漏标签
    - 点击发布后若未检测到成功信号，会直接判失败，避免误记成功

- 若后续需要重发某条内容：
  - 将对应平台的“发布状态”改回“待发布”
  - 发布器会自动解除本地防重锁
  - 不必再手动清理 `publish-ledger.json`
