# 2026-03-27 三栏桌面工作台重构

## 用户决策

- 用户明确同意：主界面可以一口气整体重做，出现问题时以 GitHub 回滚为兜底，不再要求按旧页面逐块小步验证。
- 新产品名称继续沿用：
  - 产品名：`知发`
  - 副标题：`笔记多平台发布工具`

## 新界面结构

- 主界面改为三栏桌面工作台：
  - 左栏：一级导航
  - 中栏：当前模块的设置面板 / 记录列表
  - 右栏：预览 / 运行上下文 / 数据摘要
- 左栏一级导航固定为：
  - `总览`
  - `发布`
  - `设置`
  - `数据`

## 模块拆分原则

- `总览`
  - 服务状态
  - 待发布数量
  - 下次检查时间
  - 当前任务阶段
  - 最近日志
- `发布`
  - 待处理记录
  - 非待发布记录
  - 已完成记录
  - 右侧展示当前选中记录详情
- `设置`
  - 飞书接入
  - 账号授权
  - 比特浏览器
  - 定时任务
  - 默认音乐
- `数据`
  - 配置备份导入导出
  - 完整数据导入导出
  - 本机数据路径说明

## 实现约束

- 这次重构只重做前端结构与视觉，不改现有后端 API 协议。
- 旧的 `switchTab()` 保留为兼容入口，但实际页面切换要统一走：
  - `switchSection()`
  - `switchSetupPane()`
- 右侧预览必须跟着当前一级模块 / 二级设置项动态变化，不能再是固定死区。
- 飞书配置加载后，需要同步更新：
  - 数据页路径说明
  - 左栏底部配置目录 / 数据目录

## 后续接着做时的建议

- 继续优化时，优先打磨：
  - 空状态
  - 选中态
  - 右侧预览的层级与信息密度
  - 桌面端窗口缩放时的布局表现
- 不要回退到旧的“多 tab 混排 + 单屏堆满所有设置”的结构。

## 第二轮设计修正

- 用户明确希望整体视觉更克制：
  - 白底 / 灰底为主
  - 绿色参考微信风格
  - 文字对比度必须更高
- 左栏只保留：
  - 品牌
  - 一级导航
- 左栏不应再放配置目录 / 数据目录这类路径信息。
- 中栏设计原则进一步收紧为：
  - 顶部统一的操作头
  - 下方只放可操作内容
- 纯展示型内容如：
  - 运行概况
  - 当前任务
  - 日志
  - 路径摘要
  更适合放到右侧预览区。
- 设置页二级导航需要用更大的固定卡片表达，不能只是小按钮。
