# 标题撰写规则三处口径错配复盘（2026-07-09）

> 场景：48 条北师大小学数学**授课课件**发布，标题返工 4 轮才通过。根因不是文案能力，是「SYSTEM_PROMPT 规则 / create 校验器 / 内容品类」三层口径没对齐。本文供后续单独修 `src/ai-writer.js` 的 SYSTEM_PROMPT 和 create 标题校验器用。

| 轮次 | 卡在哪 | 直接原因 |
|---|---|---|
| 1 | 标题写得像「资料/讲义」（备课少熬一晚、板书思路、一页讲透） | 子代理只能从钩子菜单①~⑧选，全是老师收益词；冷眼审查按字面放行，但不符「授课课件」 |
| 2 | dry-run 卡字数，23 条 21-22 字超 20 | 子代理自检把书名号《》排除计字，create 校验器把《》计入，每条差 2 字 |

| 4 | 用户否掉钩子方向 | 授课课件应突出「学生结果」（公式五），但钩子菜单①~⑧没有一个是学生结果类 |

## 二、三处口径错配（要改的点）

### 错配 1：钩子菜单 vs 授课课件品类（最根本，最高优先）
- **位置**：`src/ai-writer.js` SYSTEM_PROMPT「标题双层结构」钩子层封闭菜单①~⑧
- **现状**：①备课少熬一晚 ②开学第一讲 ③新接班先收好 ④板书思路都在这 ⑤衔接课备课不愁 ⑥一页讲透省时间 ⑦摸底考命题参考 ⑧晨读自习能用上 —— **全是「老师拿资料的收益/使用场景」，没有一个是「这节课这样上、学生怎么样」**。
- **矛盾**：同文件「公式五（授课课件）」明确要求 `课名 + 这样讲/上 + 学生结果`，结果对象是学生；但钩子层封闭菜单给不了学生结果，导致授课课件标题只能套老师收益钩子 → 写得像讲义。用户明确指出「板书思路」「一页讲透」不像授课课件。
- **建议**：给「授课课件 / 复习课件」品类的钩子层增设「学生结果类」钩子池（如：学生一看就懂 / 学生秒懂 / 学生不出错 / 学生跟得上），或在规则里写明「走公式五/六的标题，钩子层可用学生结果句替代封闭菜单①~⑧」。

### 错配 2：字数计数口径（校验器 vs prompt 自检不一致）
- **位置**：create 校验器 `Title length` ↔ SYSTEM_PROMPT 字数规则
- **现状**：校验器口径 = `len(title)`（emoji 计 1 字、书名号《》计入、所有字符全计）；但 SYSTEM_PROMPT 只在**标点**规则里写「不含《》和 emoji」，**字数**规则没写清《》和 emoji 算不算，子代理据此排除《》自检 → 每条比校验器少算 2 字。
- **建议**：SYSTEM_PROMPT 字数规则明确写「字数 = 含 emoji（1 字）+ 含《》+ 所有字符的总长，硬边界 10-20」，与校验器 `len()` 完全对齐。

### 错配 3：标点检查把课名括号计入
- **位置**：create 校验器标点检查
- **现状**：把课名《分数除法（一）》《小数的再认识（版本二）》里的**全角括号（）计为标点**。遇到带课时序号括号的课名，「标点≤1」必然超标；而括号是课名固有部分，「课名不缩写」规则又不许删 → 三规则打架，本次只能去掉课时序号「（一）」绕过。
- **建议**：校验器标点检查**排除书名号《》内部**的括号；或 SYSTEM_PROMPT 明确「课名（书名号内）自带的括号不计入标点数」。
