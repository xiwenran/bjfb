# 融景合成图 → 知发上传 Skill

把融景已合成的笔记图，通过知发（zhifa，运行在 localhost:3210）上传到飞书多维表格，定时发布小红书 / 抖音。

## CLI 路径

```bash
python3 /Users/xili/zhifa/scripts/skill_upload.py
```

## 输入目录结构（融景标准输出）

```
合成图/
  课件A/          ← 主题文件夹
    1/            ← 模板1（一篇笔记）
      1.jpg  2.jpg ...
    2/            ← 模板2（一篇笔记）
      1.jpg  2.jpg ...
  课件B/
    1/
      1.jpg  2.jpg ...
```

## 工作流程

### Step 0：确认参数

用户在**一条消息里**提供所有必要信息，之后只需在 Step 4 确认内容预览即可。

**标准输入格式：**

```
合成图文件夹：/Users/xili/笔记制作输出/合成图
账号：小红书-测试账号
发布时间：后天 15:00

组1：@草船借箭封面.jpg
主题：草船借箭的历史背景与战争智慧

组2：@赤壁之战封面.jpg
主题：赤壁之战：以少胜多的经典战例
```

- **合成图文件夹**：融景输出的 `合成图/` 目录路径
- **封面图**：每个主题组一张，用 @ 发送或提供文件路径；文件名无所谓，Skill 统一存为 `0.jpg`
- **主题**：每组一句话，作为 Claude 写文案的依据
- **账号**：小红书账号名 和/或 抖音账号名（可同时写两个）
- **发布时间**：支持自然语言（「后天 15:00」），转成 `YYYY-MM-DD HH:mm` 格式

### Step 1：检查知发服务

```bash
curl -sf http://localhost:3210/api/import/preflight
```

失败 → 提醒用户打开知发 App（菜单栏可见），等用户确认后继续。

### Step 2：扫描合成图文件夹

```bash
python3 /Users/xili/zhifa/scripts/skill_upload.py scan <合成图文件夹>
```

输出每个主题文件夹（topic group）下的笔记列表（每个模板子文件夹为一篇笔记）。

若发现的主题组数与用户提供的封面/主题数量不一致，停下确认——不继续执行。

### Step 3：放置封面图

对每个主题组的**每个笔记子文件夹**，把对应封面图复制为 `0.jpg`：

```bash
cp "<封面图路径>" "<笔记子文件夹>/0.jpg"
```

- `0.jpg` 排序键最小（0 < 1 < 2…），保证封面始终排第一
- 同一主题的多个模板子文件夹（`1/`、`2/`、`3/`）放同一张封面图

**如果封面图是用户通过 @ 发给你的**：读取附件的实际磁盘路径（通常在 `/var/folders/…` 或用户 Downloads 目录），直接 cp 过去。

### Step 4：Claude 生成文案

为每个主题组生成小红书文案（同一主题的多个模板共用同一套内容）：

- **标题**：≤20 字，吸引人，口语化，结尾可带 ❓💡✨ 等符号，**不要** `#话题`
- **正文**：200–400 字，有实质干货，口语化，结尾 3–5 个 `#话题标签`
- **标签**：3–5 个关键词（不带 `#`，用于飞书「标签」字段）

生成后展示预览：

```
📋 内容预览（请确认后说「确认」继续）

── 组1：草船借箭的历史背景与战争智慧 ──
标题：草船借箭，诸葛亮凭什么敢这么赌？
正文：（前 120 字）……
标签：历史故事 / 三国 / 战争智慧

── 组2：赤壁之战：以少胜多的经典战例 ──
…
```

等用户确认后才执行 Step 5。

### Step 5：构建 records 并上传

把扫描结果 + 封面 + 文案组装成 records JSON，写入 `/tmp/zhifa_records.json`，再上传：

```bash
python3 /Users/xili/zhifa/scripts/skill_upload.py create /tmp/zhifa_records.json
```

records 每一条的结构：

```json
{
  "topic": "课件A",
  "topicOverride": "草船借箭的历史背景与战争智慧",
  "noteKey": "课件A/1",
  "folderPath": "/Users/xili/笔记制作输出/合成图/课件A/1",
  "images": [
    {"name": "0.jpg", "path": "/…/合成图/课件A/1/0.jpg", "size": 12345},
    {"name": "1.jpg", "path": "/…/合成图/课件A/1/1.jpg", "size": 23456}
  ],
  "xiaohongshuAccount": "小红书-测试账号",
  "douyinAccount": "",
  "publishTime": "2026-04-30 15:00",
  "xiaohongshuChannel": "蚁小二",
  "title": "草船借箭，诸葛亮凭什么敢这么赌？",
  "description": "…（完整正文）…",
  "tags": ["历史故事", "三国", "战争智慧"]
}
```

**关键**：`title` 字段预填时，知发跳过自己的 AI 生成，直接用你写的文案。

### Step 6：报告结果

```
✅ 上传完成

处理 2 组，共 4 篇笔记：
  ✓ 成功 4 篇
  - 跳过 0 篇（已存在）
  ✗ 失败 0 篇

飞书表格已更新，可在知发里查看待发布队列。
```

## 注意事项

- 知发服务必须运行（macOS 菜单栏可见），否则第一步检查会失败
- 查重：相同内容二次上传知发会自动跳过（指纹查重），不会重复创建记录
- 发布时间用自然语言（「后天 15:00」「明天上午 10 点」）→ 转为 `YYYY-MM-DD HH:mm`
- 如果某组不需要发布到某平台，把对应账号字段留空即可
