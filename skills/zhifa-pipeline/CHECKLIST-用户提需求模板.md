## 默认行为（用户未指定时按此处理）

| 项 | 默认 | 修改方式 |
|---|---|---|
| 主题列表 | 自动从素材文件夹名读取 | 不需要单独提供 |
| 账号分组 | 优先用 CHECKLIST；未填则读取 `accounts.json.accountGroups` 推荐 | 没有命中分组时必须回问 |
| 英语类抖音 | 默认手动发布，不自动生成抖音记录 | 明确写“英语类也自动发抖音” |
| 融景模板 | 全部 22 个 | 写"只用模板 X,Y,Z" |
| 平台搭配 | **永远独立**——同一笔记不会同时发到小红书和抖音 | 不可修改（架构原则） |
| 模板分配 | 拉丁方公式分配，同账号跨主题用不同模板 | 不可修改（CRITICAL） |
| 主题分配 | 每账号覆盖全部主题，每主题最多 18 篇 | 不可修改 |
| 标题公式 | 家长会用公式 5-8（无课文名）/ 课文用公式 1-4 | 写"标题用公式 X" |
| 标题字数 | 10-20 字 | 不可修改（ai-writer 硬约束）|
| 标签数量 | 小红书 ≤10 / 抖音 ≤5 | 写"小红书标签 N 个" |
| 描述（description） | 留空 | 写"description 写：xxx" |
| AI 写作 | 主会话先按公式手写 → 失败时调 zhifa AI Provider | 写"全部用 AI 接口" |
| 发布状态 | 上传时不写（飞书可能默认填值，由用户在飞书侧管理） | 写"上传时设为 X" |
| 上传后审查 | 自动派子代理冷眼审查（标题/标签/调度合规性） | 写"跳过审查" |
| 反馈节奏 | 按阶段汇报，标题标签生成后等用户确认 | 写"全程不打扰直到结束" |
| 未安排笔记 | 归档到 已制作/未安排/[主题]/ | 写"备份到其他路径" |
| 已安排笔记 | 归档到 已制作/已安排/[平台]/[主题]/ | 同上 |

## 工作流程概览

按顺序执行（详见 SKILL.md）：

1. 扫描素材根目录 → 确认 PPT 文件 + 封面文件
2. 调用 ppt-batch-tool + rongjing 跑融景合成（pipeline.py，约 30 分钟/7 PPT）
3. rongjing 自动放置封面（--cover-source 参数）
4. zhifa scan 合成图目录
5. POST /api/import/schedule 生成调度矩阵 + 展示给用户预览
6. **主会话按公式生成标题/标签** → 展示给用户确认（必停点）
7. POST /api/import/create-records 上传（自适应分批，skill_upload.py 内置）
8. POST /api/import/archive 自动归档已安排+未安排
9. 派子代理冷眼审查 → 报告

每个阶段完成后向用户汇报进度，第 6 步必须等用户确认才继续。
