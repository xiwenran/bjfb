## 蚁小二发布规则
- 标题最大 50 字
- 正文最大 2000 字
- 云发布时 clientId 必须设为 null
- 本机发布时必须提供官方 clientId（设备号）
- 图文/视频/文章三种类型，字段要求不同
- 视频素材需包含 duration，封面需单独提供（用户自行提供，不依赖 ffmpeg）
- ⚠️ 抖音账号在蚁小二云端授权可能过期，需用户手动重新授权
- 官方 API Key 作为 Authorization 使用，当前项目不再依赖用户名密码登录链路
- ⚠️ **图片链接要求**：不能使用占位符链接（如via.placeholder.com），必须用真实在线图片或上传到蚁小二云存储

## 精确定时发布机制
- 检查到「发布时间」未到的记录 → 自动创建一次性精确任务（at，精确到分钟）
- 到时间点才提交蚁小二，误差 0 分钟
- 发布时间已到的 → 立即提交

## 技术备忘
- ⚠️ open.feishu.cn/open-apis/ 的 GET 请求会被 CDN 拦截返回 404，需用 POST
- 飞书 batch_get_tmp_download_url 每次只传1个file_token
- 飞书附件下载临时URL自带authcode，不需额外鉴权
- 蚁小二 publish-flow 不支持 prePubTime 参数
- yixiaoer-plugin 依赖 npm install --ignore-scripts（ffmpeg-static 安装失败）
- 蚁小二 API 路径: https://www.yixiaoer.cn/api
- 飞书附件 API 返回顺序可能不一致，需按文件名排序
- 当前项目音乐搜索与模板配乐走官方 getAccountMusic / music 字段链路
- 音乐库支持 `getAccountMusicCategory` 分类页，以及“最近热歌/推荐”浏览
- 账号授权状态页不能依赖不存在的 `loginStatus` 字段，当前按官方返回的 `status`、`isFreeze`、`isOperate` 判断是否在线/可用
- 标签发布现在只走官方 `tags` 字段，不再把飞书标签强行拼进正文，避免正文污染与重复展示
- 飞书标签字段需按对象/多选结构安全解析，避免出现 `[object Object]` 混入标签内容
- 标签发布前会按平台限额筛选：抖音最多 5 个，小红书最多 10 个，并先去重再按主题相关性排序
- 小红书与抖音标题发布前都会自动按 20 字限制截断，避免因标题过长导致平台任务失败；截断信息会写入日志与飞书备注
- 默认背景音乐在发布前会用官方音乐库校验；若已失效，会自动回退到热门纯音乐/轻音乐并在页面与日志中提示
- 持久化防重依赖三处：飞书平台状态、飞书备注成功标记、本地 `publish-ledger.json`
- 现在是否允许重新提交到蚁小二，以飞书里的平台状态字段为准；若用户把平台状态手动改回“待发布”，本地账本会自动解锁并允许重发
- 发布成功后会自动把蚁小二任务提交信息和可匹配到的最近发布记录摘要回写到飞书备注
- 比特浏览器配置位于 `config.json > bitbrowser.xiaohongshu`，按“小红书账号名 → browserId”映射；当前已记录 `小雨老师`
- 比特浏览器小红书发布链路现在会显式校验：
  - 未找到话题标签入口时直接失败，不再静默跳过标签
  - 点击发布后若仍停留在发布页、未检测到成功信号，则判定为失败，避免误记为成功
- 官方插件更新检查固定对比 `https://gitee.com/wangzj141/yixiaoer-skill.git`，不依赖本机插件当前 git remote；本机 remote 若不是官方仓库，只提示“需人工确认”
- ⚠️ **图片上传修复经验（2026-03-20）**：占位符图片导致发布失败，使用 https://source.unsplash.com/1080x1350/?keyword 作为高质量替代方案
- 若需重发某条记录，优先把对应平台状态改回“待发布”；无需再手动清理本地账本
